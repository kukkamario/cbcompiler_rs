| [FD-021](archive/FD-021_PARSER_AND_SPAN_PANIC_SAFETY.md) | Parser & Span Panic-Safety | 2026-06-02 | Restores the frontend's "never abort on untrusted input" contract — closes one confirmed crash and three latent panic vectors found in the post-FD-018 review. **Recursion guard:** a shared `recursion_depth` counter on `Parser` (cap `MAX_RECURSION_DEPTH = 256`) gates the two true recursion gateways — `parse_expr_bp` and `parse_type_atom`, each split into a thin guard wrapper + `_inner` so inc/dec stays balanced on every path including `?` early returns; the counter resets at each statement boundary so a `?`-aborted sub-parse can't inflate a later statement. Pathological nesting (`((((…))))`, `----…x`, `f(g(h(…)))`, `((((Integer))))`) now emits the new **E0218** `E_NESTING_TOO_DEEP` diagnostic and returns `Expr::Error`/`TypeExpr::Error` for recovery instead of overflowing the stack (was a confirmed `exit 134` abort at depth ~6000). **Pratt panics eliminated (folded, not demoted):** `infix_bp`+`binop_from` → `infix_op` (returns `(u8,u8,BinOp)`), `prefix_bp`+`unop_from` → `prefix_op` (`(u8,UnOp)`), `postfix_bp` → `postfix_op` (`(u8,PostfixKind)`); `parse_postfix` now dispatches on an exhaustive `PostfixKind` match — the two `.expect()`s and the `unreachable!` are gone, the bp/op tables can no longer drift. **`SpanExt::slice`** is now bounds- *and* char-boundary-checked (returns `""` on bad input, never panics) matching `span_slice`/`string_value::slice`. **`offset_to_line_char_col`** floors `slice_end` to the nearest char boundary so a mid-codepoint offset clamps instead of panicking. **`ast_print::print_node`** gains `MAX_PRINT_DEPTH = 512` (above the parser cap, so every real AST still prints in full) with an elision marker, since it's reachable on untrusted ASTs via `--dump-ast`. New `crates/cb-frontend/tests/deep_nesting.rs` (12 cases) + a mid-codepoint test in `cb-diagnostics`. Full `cargo test --workspace` (all suites, 0 failed) + `clippy -D warnings` green; manual 6000-deep repro now exits 1 with E0218 (`--dump-ast` exits cleanly). Verification surfaced an out-of-scope latent `BrokenPipe` panic in the driver's stdout writer under `SIGPIPE` — flagged for a possible follow-up FD, not fixed here. |

| [FD-028](archive/FD-028_SYNTAX_FIDELITY_FOR_LEGACY_CODE.md) | Syntax Fidelity for Legacy Code (`\`/`.` field access, `^` exponent, `'` comments) | 2026-06-02 | Realigns the frontend with real CoolBasic across five divergences introduced when syntax was implemented from memory. **(1+2) `\` is the `Type` field accessor, not integer division** — removed `BinOp::IntDiv`/`IrBinOp::IntDiv` and all eval/typing/lowering arms; `\` and `.` are now fully interchangeable postfix field accessors (same binding power 34, both emit `Expr::Field`, freely mixable as `a\b.c\d`). `/` is unchanged (int division for two ints, float division when either operand is `Float`). **(3) Exponent is `^`, not `**`** — new `Op::Caret` → `BinOp::Pow` (right-assoc, prefix-bp 30 preserves `-2^2 = -(2^2)`); `**` now lexes as two `Op::Star`. **(4) `'` starts a line comment** alongside `//`/`Rem`/`/* */` (`scan_line_comment_tick`, honors `preserve_trivia`). **(5) Unary `+` is absolute value** (`+x ≡ Abs(x)`, not identity) — `IrUnOp::Plus`→`IrUnOp::Abs`; interp computes width-aware abs (signed via `wrapping_abs` matching runtime `Abs` at `MIN`, unsigned identity, `Float` via `f64::abs`); const-fold folds `+` to abs; AST-level `UnOp::Plus` and sema typing (type-preserving) unchanged. `docs/cb_syntax.md` updated (§1.2 comments, §1.7 operators, §3.3 field access, §5.1 precedence, trap list). **Verification:** full workspace tests (0 failed) incl. new lexer/parser/sema/interp cases, end-to-end driver smoke test (`player\x`/`player.x`, `2^3`, `'` comment, `+(-5)→5`), and `clippy -D warnings` all green.
